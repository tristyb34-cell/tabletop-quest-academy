# Visual Studio 2026 Reference for UE5 Development

A comprehensive reference for configuring Visual Studio 2026 for Unreal Engine 5.7 C++ game development. Every setting includes its exact path so you can find it quickly.

---

## Initial Setup

### Installation

- Download Visual Studio 2026 Community (free for individuals, open source, and small teams)
- In the Visual Studio Installer, select the **"Game development with C++"** workload
- Also check **"Desktop development with C++"** for full tooling (compilers, linkers, Windows SDK)
- Individual components to add:
  - Windows SDK (latest version available)
  - C++ profiling tools
  - C++ AddressSanitizer (catches memory bugs at runtime)
  - IntelliCode (AI-assisted completions)
- Install the **"Visual Studio Tools for Unreal Engine"** extension (available built-in or via Extensions > Manage Extensions)

### First Launch Configuration

- **Tools > Options > Environment > General**: set Color theme to Dark (recommended for long sessions), configure startup behavior (choose "Open last solution" for fast resume)
- **Tools > Options > Environment > Preview Features**: enable experimental features if you want early access to new capabilities
- **Tools > Options > Environment > Accounts**: sign in with a Microsoft account to sync settings, themes, and keybindings across devices
- **Tools > Options > Environment > Fonts and Colors**: change the editor font. JetBrains Mono at 12pt is a solid choice for code readability. Fira Code and Cascadia Code are also popular.

## Editor Settings (Every Important One)

### Text Editor General

**Path: Tools > Options > Text Editor > General**

- **Sticky Scroll**: enables scope headers (function names, class declarations) that pin to the top of the editor while you scroll through long files. Enable this; it helps you keep context in large UE5 classes.
- **Track changes**: shows a colored margin on the left side of modified lines (yellow for unsaved, green for saved changes). Useful for knowing what you've touched.
- **Brace pair colorization**: matching braces, brackets, and parentheses get unique colors so you can visually identify nested scopes. Enable it.
- **Line numbers**: show line numbers in the gutter. Enable this; you'll need line numbers for debugging and compiler errors.
- **Word wrap**: wraps long lines visually instead of requiring horizontal scrolling. Personal preference, but most C++ developers leave this off since lines should be kept reasonably short.

### Text Editor > C/C++

**Path: Tools > Options > Text Editor > C/C++ > General**

- **Auto list members**: shows the autocomplete dropdown as you type. Keep enabled.
- **Parameter information**: shows function parameter hints in a tooltip when you type the opening parenthesis. Keep enabled.
- **Auto brace completion**: automatically inserts the closing `}`, `]`, or `)` when you type the opening one. Keep enabled.

### Text Editor > C/C++ > Advanced

**Path: Tools > Options > Text Editor > C/C++ > Advanced**

This is the most important settings page for UE5 development. It controls IntelliSense behavior.

**IntelliSense settings:**

- **Disable Squiggles**: set to **False**. Squiggles are your first line of error detection. Only set to True if the false positives from UE5 macros are too distracting.
- **IntelliSense Rescan Interval**: controls how often IntelliSense re-scans changed files, in milliseconds. Default is 60000ms (1 minute). Reduce to 30000ms for faster updates if your machine can handle it.
- **Member List Filter Mode**: set to **Fuzzy** matching. This makes autocomplete more forgiving with typos and partial matches.
- **Disable Aggressive Member List Filtering**: set to **False**. Aggressive filtering narrows down autocomplete results more quickly.
- **Max Cached Translation Units**: increase to **8** if you have 32GB+ RAM. Default is 2. More cached translation units means faster IntelliSense when switching between files, at the cost of memory.
- **Disable IntelliSense**: leave this **False**. Setting it to True disables all autocomplete, parameter hints, and error squiggles.

**Browsing/Navigation settings:**

- **Disable External Dependencies Folders**: set to **True**. This hides the "External Dependencies" virtual folder in Solution Explorer, which otherwise shows thousands of engine headers and adds visual noise.
- **Enable Single File Preprocess**: set to **True**. This lets you right-click a .cpp file and see the fully preprocessed output. Extremely useful for debugging UE5 macros like GENERATED_BODY(), UPROPERTY, and UFUNCTION.

### Text Editor > C/C++ > Formatting

**Path: Tools > Options > Text Editor > C/C++ > Formatting**

- **General**: enable **"Automatically format when I paste"** to keep code clean when pasting snippets.
- **Indentation**: set to **Tabs** (UE5 convention uses Tabs, not Spaces). Tab size: 4.
- **New Lines**: set **"Place open brace on new line for functions"** to **Yes**. This is Allman style, which matches UE5's coding standard.
- **Spacing**: set **"Insert space within argument list parentheses"** to **No**. UE5 style uses `Function(Arg1, Arg2)` not `Function( Arg1, Arg2 )`.

### Text Editor > C/C++ > Code Style

**Path: Tools > Options > Text Editor > C/C++ > Code Style**

- **Generated code style**: controls how Visual Studio generates code snippets and boilerplate. Adjust to match UE5 conventions.
- **Prefer braces**: set to **Always**. This ensures `if`, `for`, and `while` statements always get braces, preventing subtle bugs from braceless single-line blocks.

## IntelliSense Configuration (Critical for UE5)

### The UE5 IntelliSense Problem

UE5 projects are massive, often spanning millions of lines of code across the engine and your game modules. Default IntelliSense settings can cause slowdowns, high memory usage, or even crashes. Tuning these settings is essential.

### Recommended Settings

**Path: Tools > Options > Text Editor > C/C++ > Advanced**

- **Max Cached Translation Units**: set to **8** (default is 2). Each cached translation unit consumes RAM but dramatically improves IntelliSense speed when switching between files. Only increase if you have 32GB+ RAM.
- **Disable Database**: set to **False**. The database stores parsed symbol information for faster lookups.
- **Disable Inactive Code Analysis**: set to **True**. This saves CPU by not analyzing code inside `#if 0` or platform-specific `#ifdef` blocks that don't apply to your current configuration.
- **IntelliSense Error Limit**: keep at **1000** (default). Don't lower this or you may miss real errors.
- **Enable Single File IntelliSense**: set to **True**. This provides IntelliSense for files even when the full solution hasn't finished parsing.

### Alternative: Visual Assist (Paid Extension)

Many UE5 developers disable default IntelliSense entirely and use Visual Assist instead. Key advantages:

- Significantly faster parsing of large UE5 projects
- Native understanding of UE5 macros (UPROPERTY, UFUNCTION, UCLASS, USTRUCT, UENUM)
- Better symbol navigation and refactoring tools
- Cost: $129 one-time license

If IntelliSense is consistently slow or producing too many false-positive errors, Visual Assist is the go-to fix in the UE5 community.

### Shared Compilation State (UE5 Feature)

UE5 projects configure IntelliSense to share compilation state across translation units. In practice, this means that when you switch between .cpp files, semantic colorization and autocomplete become available much faster than doing a fresh parse of each file from scratch.

## Debugging Settings

### General Debugging

**Path: Tools > Options > Debugging > General**

- **Enable Just My Code**: **DISABLE** this for UE5 development. You frequently need to step into engine code to understand behavior or track down bugs. With "Just My Code" enabled, the debugger skips engine source, which is counterproductive.
- **Enable .natvis diagnostic messages**: set to **True**. This helps troubleshoot UE5's custom debug visualizers if they're not displaying correctly.
- **Enable Source Link support**: set to **True**. Allows the debugger to download source from source-linked symbols.
- **Require source files to exactly match the original version**: set to **False**. This allows debugging even when source files have minor differences, which can happen during engine updates.
- **Show disassembly if source is not available**: set to **True**. When you can't access the source for a particular function, you'll at least see the assembly instructions.

### C++ Debugging

**Path: Tools > Options > Debugging > C++**

- **Enable Native Edit and Continue**: **DISABLE** this for UE5. Unreal uses its own Live Coding system instead, and having Edit and Continue enabled can cause conflicts.

### Symbols

**Path: Tools > Options > Debugging > Symbols**

- **Load only specified modules**: recommended for UE5. Loading symbols for ALL modules (engine, third-party libraries, plugins) is extremely slow and uses several gigabytes of memory.
- **Specify included modules**: add your game module name (e.g., `MyGame.dll` or `MyGameEditor.dll`). You can add engine modules as needed when debugging specific engine systems.

### Natvis Files (UE5 Custom Visualizers)

UE5 ships `.natvis` files that make Unreal types readable in the debugger watch window. Without them, types like FString, TArray, TMap, and FVector show as raw memory, which is nearly impossible to interpret.

These should auto-load when you open a UE5 project. If containers are showing raw memory instead of their contents, check:

- **Tools > Options > Debugging > General** > "Use native compatibility mode" = **False**

The natvis files are located in the engine's `Extras/VisualStudioDebugging/` directory.

### Attach to Process (Important)

When the UE5 editor is already running and you want to debug your game code:

1. **Debug > Attach to Process** (shortcut: **Ctrl+Alt+P**)
2. Find **"UnrealEditor.exe"** in the process list (use the search filter)
3. Make sure the "Attach to" field says **"Native code"**
4. Click **Attach**

This is faster than launching the editor through the debugger every time, especially during iteration.

### Conditional Breakpoints

Right-click any breakpoint and select **Conditions** to set conditional breakpoints:

- Example: `Health < 0` only triggers when the Health variable goes negative
- Example: `EnemyName == "Goblin"` only triggers for enemies named Goblin
- **Hit Count**: break only after the breakpoint has been hit N times. Useful for breaking inside loops on a specific iteration.
- **Filter**: restrict the breakpoint to specific threads or processes

## Build Configuration

### Solution Configurations for UE5

UE5 projects come with several build configurations. Understanding them is important:

- **Development Editor**: the default for day-to-day development. Includes debug info, moderate optimizations, and runs within the UE5 editor. Use this 90% of the time.
- **DebugGame Editor**: slower execution but provides full debug information with no optimizations. Use when you need to step through code and see every variable without the compiler optimizing them away.
- **Shipping**: the final optimized build for release. No debug info, no editor, no development tools, maximum optimization. Only use for final packaging.
- **Development**: standalone game build without the editor attached. Good for testing how the game runs outside the editor.
- **DebugGame**: standalone game build with full debug info. Use for debugging issues that only appear outside the editor.

### Build Settings

**Path: Tools > Options > Projects and Solutions > Build and Run**

- **Maximum number of parallel project builds**: set this to match your CPU core count (e.g., 16 for a 16-core CPU). More parallel builds means faster compilation.
- **Only build startup projects and dependencies on Run**: set to **True**. This prevents VS from rebuilding the entire solution (including engine modules you haven't changed) every time you press F5.
- **On Run, when build or deployment errors occur**: set to **"Do not launch"**. You don't want to debug a stale binary when the current build has errors.

### Live Coding (Hot Reload Replacement)

UE5 uses Live Coding instead of the traditional Edit and Continue:

- In the UE5 Editor, press **Ctrl+Alt+F11** to compile your code changes without restarting the editor
- Much faster than a full rebuild; typically takes 5 to 15 seconds
- **Only works for .cpp file changes**. Header (.h) file changes require a full rebuild because headers affect multiple translation units
- Live Coding patches the running executable in memory, so the editor stays open with your current level, actors, and state intact

## Performance Settings

### General Performance

**Path: Tools > Options > Environment > General**

- **Automatically adjust visual experience based on client performance**: set to **True**. VS will reduce animations and effects if it detects slowdowns.
- **Enable rich client visual experience**: set to **True** on powerful machines. Disable on slower hardware to reduce GPU usage.
- **Use hardware graphics acceleration if available**: set to **True**. This offloads UI rendering to the GPU, keeping the CPU free for builds and IntelliSense.

### IntelliTrace

**Path: Tools > Options > IntelliTrace**

- **Enable IntelliTrace**: **DISABLE** for UE5 development. IntelliTrace records a history of debugging events, which adds significant performance overhead. UE5 projects are already heavy enough without it.

### CodeLens

**Path: Tools > Options > Text Editor > All Languages > CodeLens**

- **Enable CodeLens**: consider **disabling** for UE5 projects. CodeLens adds inline annotations showing references, changes, and test status above functions. On a massive UE5 codebase, scanning for this information adds noticeable overhead. If you find the editor sluggish, turning off CodeLens is a quick win.

### Source Control

**Path: Tools > Options > Source Control**

- **Current source control plug-in**: set to **Git** (or Perforce if your team uses it)
- **Check current branch in status bar**: set to **True** so you always know which branch you're on

## VS2026 New Features Relevant to Game Dev

### Redesigned Settings UI

VS2026 has a completely new settings experience. You can search for any setting by keyword, filter by category, and see which settings you've changed from their defaults. This makes it much easier to find the specific option you're looking for without memorizing the full Tools > Options path.

### 30% Faster Startup

VS2026 running on .NET 10 achieves startup times up to 30% faster compared to VS2022. For UE5 projects with large solutions, this translates to noticeably shorter wait times when opening your project each morning.

### Copilot Integration

Built-in AI assistant for code generation, debugging help, and code explanation. Useful for generating boilerplate code (like repetitive UPROPERTY declarations), explaining unfamiliar engine code, and suggesting fixes for compiler errors. Not a replacement for understanding the engine, but a productivity boost.

### Profile with Copilot

Right-click a test in Test Explorer to profile it with AI-assisted analysis. Copilot can help interpret profiling results and suggest optimization strategies.

### Single File Preprocess

Right-click any .cpp file in Solution Explorer and select "Preprocess to file" to see the fully preprocessed output. Essential for debugging UE5 macros like GENERATED_BODY(), UPROPERTY, UFUNCTION, and others that expand into complex generated code.

### Improved CMake Support

Full support for CPU Usage, Events Viewer, memory usage, and File IO tools in CMake-based projects. While most UE5 projects use the .sln/.vcxproj format, some custom tools and plugins use CMake.

### Proxy Settings

New dedicated proxy settings page at **Tools > Options > Environment > Proxy Settings** for teams working behind enterprise firewalls or VPNs. Consolidates proxy configuration that was previously scattered across multiple locations.

## Troubleshooting

### IntelliSense Not Working

1. Close Visual Studio completely
2. Delete the **`.vs`** folder in your project directory (this contains the IntelliSense database and cached state)
3. In the UE5 Editor, go to **File > Refresh Visual Studio Project** to regenerate the .sln and project files
4. Reopen the .sln file in VS2026
5. Wait for IntelliSense to rebuild. On large UE5 projects this can take several minutes. The status bar at the bottom shows progress.

### Build Errors After Engine Update

1. Close both Visual Studio and the UE5 editor
2. Delete the **Binaries** folder in your project directory
3. Delete the **Intermediate** folder in your project directory
4. Delete the **`.vs`** folder
5. Regenerate project files from the UE5 editor (or right-click your .uproject file and select "Generate Visual Studio project files")
6. Open the new .sln and rebuild

### Debugger Can't Find Symbols

- Go to **Tools > Options > Debugging > Symbols**
- Add your project's `Binaries/Win64/` folder to the symbol search paths
- Make sure you're building **Development Editor** or **DebugGame Editor**, not Shipping (Shipping strips debug symbols)
- If symbols still won't load, try Debug > Windows > Modules to see which modules have symbols loaded and which don't

### Red Squiggles Everywhere But Code Compiles Fine

These are IntelliSense parser errors, not actual compiler errors. Very common with UE5 because the IntelliSense parser doesn't fully understand all UE5 macros and code generation.

- Usually harmless and can be ignored
- If they're too distracting: **Tools > Options > Text Editor > C/C++ > Advanced > Disable Squiggles = True** (nuclear option, removes all squiggles including real errors)
- Better fix: regenerate project files and wait for IntelliSense to fully reparse the solution
- Best fix: use Visual Assist, which has better UE5 macro comprehension and produces fewer false positives
