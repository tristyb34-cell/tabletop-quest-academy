## Official Epic Games Resources

### UE5 C++ Programming Documentation
- **URL**: [https://docs.unrealengine.com/5.4/en-US/programming-with-cplusplus-in-unreal-engine/](https://docs.unrealengine.com/5.4/en-US/programming-with-cplusplus-in-unreal-engine/)

### Unreal Engine API Reference
- **URL**: [https://docs.unrealengine.com/5.4/en-US/API/](https://docs.unrealengine.com/5.4/en-US/API/)
- **Description**: The full API reference for every class, function, and type in the engine. When you need to know what parameters a function takes or what a class inherits from, this is where you look.

### Gameplay Framework Overview
- **URL**: [https://docs.unrealengine.com/5.4/en-US/gameplay-framework-in-unreal-engine/](https://docs.unrealengine.com/5.4/en-US/gameplay-framework-in-unreal-engine/)
- **Description**: Explains the relationship between GameMode, GameState, PlayerController, Pawn, Character, and PlayerState. Essential reading for understanding how UE5 structures gameplay in C++.

### UE5 Coding Standard
- **URL**: [https://docs.unrealengine.com/5.4/en-US/epic-cplusplus-coding-standard-for-unreal-engine/](https://docs.unrealengine.com/5.4/en-US/epic-cplusplus-coding-standard-for-unreal-engine/)
- **Description**: Epic's official coding standard. Covers naming conventions (the A/U/F prefixes), formatting rules, and best practices. Follow this standard so your code is consistent with the engine and other developers' expectations.

## Gameplay Ability System (GAS)

### Tranek's GAS Documentation
- **URL**: [https://github.com/tranek/GASDocumentation](https://github.com/tranek/GASDocumentation)
- **Description**: The definitive community guide to the Gameplay Ability System. GAS is Unreal's framework for abilities, attributes, and gameplay effects (think RPG stats, cooldowns, buffs, debuffs). The official documentation is sparse, so Tranek's guide has become the standard reference. It is thorough, well-organized, and includes code examples. If you plan to build any RPG or ability-based game, this is essential reading.

### GAS Companion Plugin
- **URL**: [https://github.com/GASCompanion/GASCompanion-Plugin](https://github.com/GASCompanion/GASCompanion-Plugin)
- **Description**: An open-source plugin that simplifies GAS setup and provides Blueprint-friendly wrappers. Good for understanding GAS patterns even if you do not use the plugin itself.

- **Description**: Lightweight alternative. Install the C/C++ extension and configure include paths to point to your engine installation. Less feature-rich than Visual Studio or Rider for Unreal work, but usable for smaller projects.

## Tips for Learning UE5 C++

1. **Learn Blueprints first.** Understanding Blueprint concepts (actors, components, events, spawning, communication) makes C++ easier because the same concepts apply. You are just expressing them differently.

2. **Start with the macros.** UCLASS, UPROPERTY, UFUNCTION, and GENERATED_BODY are the first things to understand. Without them, nothing works.

3. **Read engine source code.** Unreal's source is available on GitHub (with an Epic account). When you want to understand how something works internally, read the source. It is the ultimate documentation.

4. **Use Live Coding for fast iteration.** Ctrl + Alt + F11 recompiles changed code without restarting the editor. It is not perfect, but it saves significant time during development.

5. **Build one system at a time.** Do not try to learn everything at once. Build a health component. Then a damage system. Then an inventory. Each project teaches you new patterns and reinforces old ones.
